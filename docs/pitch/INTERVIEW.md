## The 90-second version (if you only get a minute)

> "It's a self-hosted autonomous coding assistant. The interesting part is the economics: instead of paying a frontier model — Claude, GPT — premium rates for *every* token, the premium model only **orchestrates**. A crew of 11 specialized agents does the actual work on cheap models, under governance, with memory that compounds. So it's cheaper, it's auditable, and it gets better the more you use it. The whole thing is measured — I can show you exactly when the cheap crew is driving versus the expensive one."

## The three-beat story

1. **Problem** — a single frontier model is expensive, opaque, and forgetful.
2. **Solution** — thin premium orchestration + a cheap, governed, self-improving crew (Quark routing, WorfGate, RAG), portable across tools via MCP.
3. **Proof** — cost is measured on a ledger; every action is audited; a full deliberation costs ~$0.002–0.003.

**The one line to land:** *"Keep the expensive model thin; make the cost of every decision visible."*

**The honest line that earns trust:** *"It runs today; the shadow test that gates full autonomy is defined — and not yet cleared. I can tell you exactly what that bar is."*

## Likely questions → crisp answers

- **"Why not just use Claude Code / Copilot?"** — You still *use* the frontier model, but only as a thin orchestrator. The win is delegating the token-heavy work to a cheap, governed, self-improving crew — and **measuring** the split. Those tools bill every token at premium rates; here the premium lane stays thin.
- **"How do you know it's actually cheaper?"** — The control-lane ledger records crew-vs-premium spend per run (`pnpm lanes`), with savings computed against a frontier baseline. It's reported, not promised.
- **"Is it in production?"** — Not yet — it runs and the crew deliberates and writes code today. The go-criterion for fully replacing the premium driver is a documented shadow test (auto-recovery + cost thresholds) that isn't cleared yet. Reference clients in the docs are illustrative.
- **"How is autonomy safe?"** — WorfGate: every credential is brokered (never logged), and every file/shell op is classified green/yellow/red and **auto-remediated** (paths clamped, `--force` downgraded) rather than just blocked. Everything is audited.
- **"What did you build recently?"** — A data-backed crew stand-up; fixed two silent-degradation bugs (a fragile parser and a dead model slug); shipped async-status (live progress on every prompt, with a timeout guardrail after a real invisible hang); and a reusable presentation system that pitches any client/project/story — including this deck, which the crew authored.

Shadow test is a criterion, not a passed result · cost figures are per-deliberation micro-costs, not audited aggregates · no verified production customers yet · the README still describes the older Aha product (crew identity is in CLAUDE.md).
