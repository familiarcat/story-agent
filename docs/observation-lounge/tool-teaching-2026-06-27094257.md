## Goals

GOALS:  
1. Define the TEACHING PROTOCOL: Specify the RAG tool-card shape and tags for crew-wide tool knowledge sharing.  
2. Ensure PROPAGATION + RECALL: Crew members efficiently recall peer-taught tools without duplication.  
3. Maintain TRUST: Shared tools remain human-gated for execution; tool-cards include gate status.  

CONCEPTS:  
- Tool-card shape: name, category, capabilities, invocation, usage context, human-gate status, owning role.  
- Tags: role categories, tool name, "tool-card".  
- Recall: `rag_recall` over tags + `getApprovedToolsForCrew`.  
- Trust: Tool-cards include `autoExecute: false` and require re-check.  
- Cost: Registry queries and deliberations are avoided post-teaching.
