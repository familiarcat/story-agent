## DECISION GATES UNDER THIS MODEL

### Riker — GREEN Gate (No Approval, Just Do It)
- Daily standup facilitation
- Task reallocation / pairing assignments
- Scope clarification (based on existing docs)
- Blocker troubleshooting (<24h)
- Work prioritization within sprint

**Decision Time:** <5 min (Riker autonomous)

### Picard — YELLOW Gate (Light Approval)
- Sprint goal changes
- Scope adjustments (>10% of work)
- Crew reallocation (cross-team impact)
- Timeline concerns (will we hit Day 10?)
- Crew health concerns

**Decision Time:** <30 min (Picard in loop)  
**Example:** "Quark's cost ledger is too complex. Reduce scope by 2 pts?"

### Admiral — RED Gate (Your Approval)
- Budget override (spending >$5K policy limit)
- Security/policy exception (library licensing, compliance)
- Client communication/approval
- Release timeline changes (>2 days slip)
- Major feature trade-offs

**Decision Time:** <4 hours (approval loop + crew execution)  
**Example:** "Cost ledger hit budget. Override?"
