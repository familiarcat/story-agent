## 🧠 Crew Learning Architecture

Each crew member can store **personal memories** in Supabase:

```typescript
// Example: Worf's RLS Learning
await storeCrewPersonalMemory({
  crew_id: 'worf',
  memory_type: 'lesson_learned',
  title: 'RLS Composite Key Pattern for Multi-Tenant Systems',
  content: 'Discovered that org_id must be first column in composite keys...',
  project_id: 'client-pctms',
  task_id: 'CLIENT-001',
  tags: ['rls', 'security', 'multi-tenant'],
});

// Later, on new project, retrieve that learning
const worf_rls_insights = await searchCrewPersonalMemories('worf', 'RLS pattern');
```

### Memory Types
- **insight** — Discovery or realization
- **lesson_learned** — Learning from experience or mistakes
- **decision_note** — Rationale for architectural decisions
- **reminder** — Procedural reminders for future tasks

## 👨‍💼 How They Work Together

### Example: Multi-Tenant Database Design Task

**Task**: Design multi-tenant data isolation for new client project

**Crew Assignment** (automatic via domain routing):
1. **Worf** (Primary) — Security policies and RLS
2. **Data** (Primary) — Schema design consistency
3. **Geordi** (Secondary) — Performance baseline
4. **Riker** (Secondary) — Implementation sequence
5. **O'Brien** (Secondary) — Integration testing

**Execution Flow**:
1. **Worf** retrieves his RLS learnings from past projects
2. **Data** references past schema patterns and migration strategies
3. **Geordi** retrieves performance baselines for similar patterns
4. **Riker** sequences the work to catch issues early
5. **O'Brien** validates integration with deployment pipeline

**Knowledge Transfer**:
Each crew member stores new learnings for future projects:
- Worf documents the specific RLS pattern used
- Data documents schema decisions and versioning
- Geordi documents performance baselines
- Riker documents implementation sequence
- O'Brien documents integration testing strategy

### Future Projects Benefit

When the next client project starts and needs multi-tenant design:
- **Same crew is routed** automatically (domain expertise)
- **All past learnings are immediately available** (personal memories)
- **Implementation time drops 80%** (from 2 weeks to 2 days)
- **Quality stays high** (proven patterns are reused)
